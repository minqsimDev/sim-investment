# 시장 페이지 — 레짐 주역 재설계 (설계)

작성일: 2026-07-01

## 배경 / 문제

시장 페이지는 `요약 + 자산군 7탭(us·kr·crypto·etf·commodities·fx·rates)`의 탭 라우터다. 각 자산군은 지표·표·히트맵을 나열한다. 문제: **다른 앱(야후·네이버·토스)과 똑같이 "숫자를 보여주는" 데서 끝나** SIM의 정체성(관점·해석)이 드러나지 않는다.

## 북극성 / 목표

SIM의 차별점 = **"조회 도구가 아니라 관점·해석·행동을 주는 개인화 코치"**. 시장 페이지도 지표 나열이 아니라 **"오늘 시장을 해석(레짐 판정 + 왜)하고, 그게 내 노출에 뭘 의미하는지 가볍게 연결"** 해야 한다. → **요약(기본 진입) 화면을 레짐 내러티브로 재설계.**

## 비목표 (YAGNI)

- 자산군 7탭 *내부* 재설계 — 이번 범위 아님(기존 모듈 그대로 드릴다운으로 재사용).
- 시장 탭 가지치기 — 사용률 계측(진행 중) 데이터 확인 후 별도 과제.
- 신규 레짐/신호 산식 — 신설 금지. 기존 `compute_regime_signals`·`_compass_model` 재사용(SSOT).
- 시장 페이지를 개인화 대시보드로 바꾸는 것 — 시장=general 원칙 유지([[feedback_market_vs_portfolio_ia]]). 개인화는 "가벼운 브리지 1줄"만.

## 설계

### 1) 정보구조 · 스코프
`요약 + 자산군 7탭` 구조 유지. **요약 화면(`_market_summary_all` 의 진입 뷰)을 레짐 내러티브로 재구성.** 자산군 7탭은 판정의 근거이자 드릴다운으로 역할 재정의(모듈 재사용).

### 2) 요약 화면 레이아웃 (레짐이 얼굴)

위→아래 4블록:

1. **레짐 헤드라인 (주역)**
   - "오늘 시장 · **방어 모드**"(또는 위험선호/혼조) 큰 판정 + 한 줄 해석.
   - 데이터: overview 의 `_compass_model(sig_map, btc_chg, kweb_chg)` 재사용 → (direction, note, tone) 그대로. 전체현황 나침반과 **동일 판정**(앱 전체 일관).
   - 톤 색: 중립 → 골드 → 앰버(위험경고) 단계. 손익 빨강/파랑과 분리(색체계 준수).

2. **왜 (판정 근거 신호 스트립)**
   - 레짐을 만든 핵심 신호(금리·달러·모멘텀·breadth 등)를 *위험/완충* 태그 칩으로 나열 — "이 판정이 왜 나왔나".
   - 데이터: `compute_regime_signals`(overview `_cached_regime_signals` 캐시) 재사용 — 리스크 페이지 국면 신호와 동일 소스(SSOT). 신호명은 기존 한글 매핑 사용.

3. **자산군 한눈**
   - 지수·환율·원자재·크립토 방향 요약 카드. 기존 `시장 한눈`(mg-card, `_market_snapshot_cards`) 을 재사용·정리해 레짐 헤드라인 아래로 배치.
   - 각 카드 클릭 → 해당 자산군 탭으로 이동(기존 `data-mkt-tab` 라디오 위임 브리지 그대로).

4. **가벼운 내 노출 브리지 (1줄)**
   - "이 국면에서 내 노출은? → 리스크 진단" 정도의 **링크 1줄**. 시장=general 유지, 상세·개인화는 포트폴리오 리스크 탭이 담당.
   - 로그인 시에만 노출(게스트/미연결은 생략). 링크 suffix 는 기존 세션 토큰(`_market_suffix`) 사용.

### 3) 색 · 일관성
레짐 방향 톤 = 전체현황 나침반·리스크 국면과 동일(중립/골드/앰버). "블랙박스 아님" 원칙에 맞춰, 근거 신호 스트립이 판정의 근거를 그대로 노출.

### 4) 자산군 탭
현 7탭 모듈(`us_stocks`·`kr_stocks`·… `embedded=True`) 그대로 드릴다운. 변경 없음.

## 영향 범위 / 파일
- `ui/pages/market.py` — `_market_summary_all` 진입 뷰 재구성(레짐 헤드라인 + 근거 신호 + 시장 한눈 + 브리지). 레짐 판정·신호는 import 재사용, 신설 없음.
- **공용 레짐 헬퍼 추출**: `_compass_model`·`_cached_regime_signals`(현재 overview.py 내부)를 새 공용 모듈 `ui/components/regime.py`로 이동하고, overview·market 양쪽이 import(단일 출처, 순환 import 회피). 판정 로직 변경 없이 위치만 이동. overview 는 새 모듈에서 import 하도록 수정.
- CSS: `_MARKET_LOCAL_CSS`(market.py) 에 레짐 헤드라인·신호 칩·브리지 스타일 추가. 기존 mg-card 재사용.

## 검증
- 요약 진입 시 레짐 헤드라인이 최상단, 판정이 전체현황 나침반과 동일(같은 데이터).
- 근거 신호 칩이 리스크 페이지 국면 신호와 값 일치.
- 시장 한눈 카드 클릭 → 자산군 탭 이동 정상(기존 브리지).
- 브리지 1줄: 로그인 시만, 클릭 → 포트폴리오 리스크 탭. 게스트 미노출.
- 색: 레짐 톤 = 중립/골드/앰버(손익색 미사용). 데스크톱·모바일.

## 후속 (범위 밖)
- 자산군 탭 내부 재설계, 시장 탭 가지치기(사용률 데이터 후).
