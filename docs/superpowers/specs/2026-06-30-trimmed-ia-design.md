# 트림된 IA — 중복 제거·직관화 (설계)

작성일: 2026-06-30

## 배경 / 문제

기능이 늘며 화면 간 책임이 겹쳤다. 구체적 중복:

- **홈 = 전체현황**: `app.py:_home_render()` 가 `overview.render()` 를 그대로 호출 — 동일 페이지가 네비에 2개.
- **리스크 콘텐츠 3중 노출**: PB 진단·벤치마크·집중도가 전체현황(요약) + 포트폴리오(풀) + 리스크(풀) 에 분산.
- **시장 9탭**: 요약·전체·US·KR·크립토·ETF·원자재·환율·금리 — 사용률 불명.

## 북극성

**"시장 조망 + 내 자산 균형"** (대시보드 중심). 전체현황이 허브, 시장은 광범위 조망, 포트폴리오는 내 관점 + 위험.

## 목표

겹치는 화면을 합치고 각 화면에 단일 책임을 부여해, 네비를 3개로 줄이고 직관적인 핵심만 남긴다.

## 비목표 (YAGNI / 손대지 않음)

- **레거시 라우트 스텁**(`/movers·/us-stocks·/kr-stocks·/commodities·/fx` → `market.render()`): 북마크 보호용·네비 비노출. 유지.
- **게스트 전용 렌더**(`portfolio_guest.py`): 데모 목적. 유지.
- **시장 탭 모듈 파일**(`us_stocks.py·kr_stocks.py·commodities.py·fx_rates.py·crypto.py·etf.py·major_movers.py`): `market.py` 가 탭 렌더에 재사용 중 — 죽은 코드 아님. 유지.
- **시장 탭 가지치기**: 사용률 데이터 없이 판단 보류 → 후속 과제(계측 후 결정). 이번 범위에선 9탭 그대로.
- 시각 디자인 대개편·신규 기능: 범위 밖. 이번엔 구조(IA) 정리만.

## 설계

### 1) 화면 역할 (단일 책임)

| 화면 | 라우트 | 역할 | 데이터 깊이 |
|---|---|---|---|
| **전체현황** (기본 랜딩) | `/overview` (+`/home` 흡수) | 요약 허브 — 시장 다이제스트 + 내 자산 요약 + 위험 1줄 | **요약만**, 상세는 드릴다운 링크 |
| **시장** | `/market` | 광범위 시장 조망(보유 무관, 일반 유니버스). 9탭 현행 | 시장 데이터 |
| **포트폴리오** | `/portfolio` | 내 보유 + 자산여정 + 리스크 진단. 리스크의 **정본** | 내 보유 상세 |

### 2) 네비 3개로 축소

- **홈 제거**: `_home_render` 가 `overview.render()` 호출하는 순수 중복. `/home` 은 라우트로 남겨 전체현황을 렌더(북마크 보호)하되 네비에서 제외하고, **전체현황(`/overview`)을 기본 랜딩(default)** 으로 승격.
- **리스크 네비 제거**: `risk_signals` 콘텐츠를 포트폴리오로 이전. `/risk` 라우트는 **포트폴리오 페이지를 렌더(리스크 진단 탭 활성)** 하도록 스텁화 — 북마크/딥링크가 깨지지 않게(레거시 스텁과 동일 패턴, 리다이렉트가 아니라 같은 페이지 렌더).
- 남는 네비(custom nav 핀 + 숨은 `st.page_link` 양쪽): **전체현황 · 시장 · 포트폴리오**. `dash_style.py` 의 네비 핀 배열·`page_link` 순서·JS 브리지 인덱스 매핑을 3개 기준으로 갱신.

### 3) 포트폴리오 = 내 보유 + 리스크 서브탭

포트폴리오 페이지 상단에 `st.tabs(["내 보유", "리스크 진단"])`:

- **내 보유 탭**: 자산여정(배지·진행률 바·카드·설정) + 자산배분 트리맵 + 보유표 + 스크린샷 갱신.
- **리스크 진단 탭**: PB 진단 풀카드(집중·충격·재배분·벤치마크) + 정량지표(베타·HHI·변동성) + 시장신호→노출 매핑 + 텔레그램 알림 연결.

`risk_signals.render()` 의 구성요소를 포트폴리오 리스크 탭으로 이식하되, 진단/벤치 계산은 기존 단일 출처(`_account_diag`)를 공유해 중복 계산을 만들지 않는다.

### 4) 중복 제거 규칙 — 정본 1곳

- PB 진단·벤치마크·집중도의 **정본 = 포트폴리오(리스크 탭)**.
- 전체현황은 **요약 1줄(+상세 링크)** 만 노출(`_pb_risk_summary_html` 유지), 풀카드 미표시.
- 자산배분·수익률 등도 동일 원칙: 전체현황은 요약/미니, 상세는 포트폴리오.

### 5) 라우팅·세션

- 기본 랜딩을 `/overview` 로(현재 `/home` default). 로그인 직후·`_user`/`_auth` 세션 복원 진입점도 전체현황으로 일관.
- `/home`, `/risk` 는 라우트 유지(각각 전체현황·포트폴리오 렌더)해 기존 북마크·딥링크가 깨지지 않게 한다.
- 네비 링크 생성부(`search·dash_style·market·overview·stock_detail·portfolio`)의 세션 토큰 suffix 로직은 그대로(보안 수정 반영본 유지).

## 영향 범위 / 파일

- `app.py` — 네비 페이지 목록 3개로(홈·리스크 비노출), 기본 랜딩 `/overview`, `/risk`→포트폴리오 흡수 스텁.
- `ui/components/dash_style.py` — 네비 핀 3개, 숨은 `page_link` 순서·JS 브리지 인덱스 매핑 갱신.
- `ui/pages/portfolio.py` — 상단 `내 보유` / `리스크 진단` 서브탭 도입, 리스크 콘텐츠 이식.
- `ui/pages/risk_signals.py` — 렌더 본문을 재사용 가능한 함수로 추출(예: `render_risk_body(...)`). 포트폴리오의 리스크 탭과 `/risk` 라우트 스텁이 **동일 함수**를 호출 → 콘텐츠 중복 없음. `render()` 는 그 함수를 부르는 얇은 래퍼로 남긴다.
- `ui/pages/overview.py` — 풀 리스크 카드가 있으면 요약으로 강등(요약+링크만).

## 검증

- 네비에 전체현황·시장·포트폴리오 3개만 노출, 핀↔실제 라우팅 1:1(JS 브리지 인덱스 정상).
- 기본 진입(`/`) = 전체현황. `/home`·`/risk` 북마크 → 각각 전체현황·포트폴리오 정상 렌더(Page not found 모달 없음).
- 포트폴리오 `내 보유`/`리스크 진단` 탭 전환 정상, 리스크 탭 내용이 옛 리스크 페이지와 동등.
- PB 진단·벤치마크가 전체현황엔 요약만, 포트폴리오 리스크 탭엔 풀카드(중복 없음). 계산은 단일 출처 동일값.
- 데스크톱·모바일 레이아웃, 게스트 경로 정상.

## 후속 과제 (이번 범위 밖)

- (선택) 보유·텔레그램 chat_id 필드 단위 암호화.
