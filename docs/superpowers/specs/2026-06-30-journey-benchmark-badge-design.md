# 자산 여정 — 벤치마크 기준 상태 배지 (설계)

작성일: 2026-06-30

## 배경 / 문제

포트폴리오 페이지 "자산 여정" 섹션의 우측 상단에는 상태 배지 2개가 있다.

- **단계(phase) 배지** `순항 중` — 진행률(`progress_pct`)만으로 결정되는 위치 라벨(초반/순항/막바지/도달).
- **페이스(pace) 배지** `목표 페이스 크게 하회` — 목표 금액을 목표일까지 달성하는 데 필요한 CAGR 대비 실제 CAGR 비교.

두 배지가 나란히 있어 `순항 중`(초록=양호)과 `목표 페이스 크게 하회`(주황=경고)가 **서로 모순**돼 보인다("잘 가는 건가 못 가는 건가"). 또한 배지 줄이 `justify-content:flex-end`라 아래 2×2 카드 그리드의 열 경계선과 정렬이 안 맞고, 모바일에서 좁게 보인다.

## 목표

상태 배지를 **"같은 기간 시장(내 보유 카테고리 벤치마크)을 이겼는가"** 기준의 **단일 배지**로 통합한다. 모순 해소 + 정렬 정리 + 모바일 가독성을 함께 처리한다.

## 비목표 (YAGNI)

- 목표 도달률 헤드라인·진행률 바·4개 카드(목표까지/예상기간/연성장률/현재자산)는 **그대로** 둔다. 이들은 *목표 도달* 계산이고, 배지는 *시장 대비* 평가로 역할을 분리한다.
- 종목별(개별 티커) 벤치마크는 만들지 않는다. 카테고리 단위까지만.
- `core/journey.py`의 `pace_months`/`stage_label`은 다른 화면에서 쓰이지 않으므로 함수 자체는 남겨두되, 자산 여정 배지에서는 더 이상 사용하지 않는다.

## 설계

### 1) 블렌드 벤치마크 (core/pb.py)

카테고리 → 벤치마크 티커 매핑:

| 카테고리 | 벤치마크 티커 |
|---|---|
| 미국주식 | SPY |
| 국내주식 | ^KS11 |
| ETF | 통화로 구분 — USD→SPY, KRW→^KS11 |
| 원자재 | GC=F |
| 크립토 / 코인 | BTC-USD |
| 채권 | AGG |
| 현금 | (벤치 없음, 수익률 0%) |

신규 함수 `blended_benchmark_return(positions, start_date, today=None) -> dict`:

- 각 보유의 평가금액(`market_value`)을 비중으로 사용(현금 포함 총액 기준).
- 필요한 distinct 티커들의 동일 기간(`start_date`~today) 누적 수익률을 받아온다 — 기존 `bench_returns`가 쓰는 `data.session.cached_download` 경로 재사용(가격 SSOT 유지).
- `blended = Σ(비중ᵢ × 카테고리 벤치 수익률ᵢ)` (현금 비중은 0% 기여).
- 다운로드 실패 티커가 있으면 그 비중만큼은 제외하고 커버된 비중으로 재정규화.
- 반환: `{"blended": float(소수), "coverage": float(0~1, 벤치 커버된 비중), "by_cat": {...}}`. 커버리지가 너무 낮거나(<0.5) 데이터 없으면 호출부에서 배지 숨김.

`bench_returns`의 인덱스 수익률 계산 로직을 재사용할 수 있도록, 티커 집합 → {ticker: 수익률} 내부 헬퍼로 추출(중복 제거). 기존 `bench_returns(start_date)` 공개 시그니처는 유지.

### 2) 분류 (ui/pages/portfolio.py)

`excess = my_return − blended` (둘 다 기간 누적 %p):

| 조건 | 배지 문구 | 색(클래스) |
|---|---|---|
| excess ≥ +20%p | `시장 크게 상회 +N.N%p` | 골드 (ahead) |
| +5 ~ +20%p | `시장 상회 +N.N%p` | 골드 (ahead) |
| −5 ~ +5%p | `순항 · 시장 수준` | 중립 (ontrack) |
| −20 ~ −5%p | `목표 페이스 하회 −N.N%p` | 주황 (behind) |
| ≤ −20%p | `목표 페이스 크게 하회 −N.N%p` | 주황 (behind) |

- 임계 밴드: **±5%p**(순항 밴드), **±20%p**(크게). 상수로 분리해 조정 용이.
- 배지에 실제 격차 `±N.N%p`를 함께 표기해 "얼마나" 앞/뒤인지 보이게 한다.
- `my_return`은 기존 `_account_diag`/`pb_diagnostics`와 동일 출처(원가 역산 기준)로 계산해 화면 간 일관성 유지.
- 데이터 없음/커버리지 부족/`start_value` 0 → 배지 미표시(빈 문자열). 기존 graceful 폴백 유지.

색 의미는 메모리 색 시스템과 일관: 골드=강조(우수), 주황=경고(부진), 중립 회색=중립. 손익 빨강/파랑과 분리.

### 3) 정렬 + 모바일 (portfolio_css.py)

- 배지가 1개로 줄어 우측 카드 그리드 우측 끝선에 깔끔히 정렬(라인 어긋남 해소). `aj-badgewrap`은 단일 배지 우측 정렬 유지 + 카드와의 하단 간격 정리.
- 모바일: `aj-cards` 2×2 유지하되 좁은 폭(≤480px)에서 폰트/패딩 축소, 배지 줄바꿈 안전 처리(`white-space`/`flex-wrap`).

## 영향 범위 / 파일

- `core/pb.py` — 카테고리→벤치 매핑, `blended_benchmark_return`, 티커 수익률 내부 헬퍼.
- `ui/pages/portfolio.py` — `_render_asset_journey` 배지 로직 교체(phase+pace → 벤치마크 단일 배지). 단일 배지로 줄어든 우측 컬럼 렌더 정리.
- `ui/pages/portfolio_css.py` — `aj-badgewrap`/`aj-cards` 정렬·모바일.

## 검증

- 보유 조합(미국주식 위주 / 크립토 포함 / 채권 포함 / 현금 비중 큼)별로 블렌드가 합리적인지, 커버리지 재정규화가 맞는지 수동 확인.
- 내 수익률이 벤치보다 위/비슷/아래 3가지 케이스에서 배지 문구·색이 의도대로 나오는지.
- 데이터/다운로드 실패 시 배지가 안전하게 숨는지.
- 데스크톱·모바일 폭에서 정렬·가독성.
